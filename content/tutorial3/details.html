Navigate to this element We saw how easy it was to import and use a custom element, but now we will see how to interact with the element, feed it some data and get some data back

First, lets have a look at the element we have so far, its a nice simple example, but not really real life example.

Custom element should be a bit customized, and it should encapsulate some sort of reusable fuctionality

Looking at our example, its does indeed represent our data in a nice way, but as long as the data is contained inside the element, there is little to this element

To pass data in (and out) of your element, you have two ways, depending on what sort of data

  • Data Binding: if you need to pass row data that you will handle inside the element, data binding is the way. You can pass any JS friendly data form, including Object
  • using slot: if you need to pass entire html content to be represented inside the custome element (like our example), then you can use this

Our element has two main parts: the main part where the main description goes, and the details popup modal where all the details are explained. We have the custom element taking care of the styling and the modal window creation/setup, while the user of this element can still feed it with whatever data he might like. This is done using slot tag: in the element we just use:

  .

<div class="w3-container">

  <slot name="general-desc"></slot>

  <div class="w3-row">

  .

When we use this element, its up to the user to provide the contents of this slot:

<p slot="general-desc">general stuff</p>

the content of slot does not have to be a text, whatever you pass with the attribute slot will be allocated in that section:

<div slot="details-modal">

  <h3>Must reads</h3>

  .

  .

</div>

This shows how to controle the contents of the element, but thats not always enough, say that we need to controle the text in the button that will open the details modal, passing that as html wont work as that part of the element is not exposed a a slot, so we will have to find another way: Data bindings

In the script section, when we invoke the Polymer function to setup our element, we can define properties, those are properties of the custom element, which can be any object.

Property value can be set by many means, refer to the polymer documentation for details, here we are setting the propery value when we use the element in our wrapper page:

<waffi-element-3 button-text="Custom Button Text">

In the element definition, we can use this value to fill the text for the button:

<p><button class="w3-button w3-padding-large w3-white w3-border" onClick="[[_openModal]]"><b>[[buttonText]]</b></button></p>

Also note that we set a default value for this property, so the default text will be shown if the user does not specify anything
