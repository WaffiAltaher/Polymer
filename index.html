NodeJS

What is it?

NodeJS is a runtime for Javascript. This means it is used to run JavaScript on the server side, so you can create backend applications, or REST APIs using Javascript.

So why do I need it in a front end project

Any decent site will consist of more than just static data, this means that chances are your web app will be using some form of API to retrieve data from your backend. Now you still wont need NodeJS for that, but in order to be able to test your application, you will either need to depend on your backend appliction(s), or you need to be able to quickly spin a server that can serve your web application and stub/mock your APIs. This is were NodeJS comes into play.

NPM

NPM for NodeJS is like maven for Java development, it is the package manager that allowes you to define the models that your server side code needs and it will download it automaticaly for you.

examples of stuff that you will need to download via npm are bower, karma, protractor, http-server, etc

*if you dont know what are those, dont worry, some of them are mentioned below, some will be explained later on

Bower

Again, bower is like the maven of Javascript. So how is it different than npm?

npm is for components that you will need in the server side that you will start while you are testing your application, while bower is used for dependencies that you wanna use in the browser, stuff like you polymer library, your css libraries and the likes.

Grunt

Grunt is a task runner that you use it to automated repeated task, kind of a miniture Jenkins. For example, every time you want to execute all your tests, then you have to issue some commands to start your local nodeJS http server, startup a test suite and maybe upload the test report somewhere, you can configure a grunt job to do all of this for you.

Another example is if you use a typed javascript variation, then everytime before you run your app you must compile your typescript into javascript.

Sitepoint Tutorial

This one-page simple tutorial is quite old (Polymer 0.3.4), but the concepts there are explained nicely, use it for understanding the concepts rather than learning exact syntax

Also worth noting is that the tutorial explains the /deep/ and ::shadow selectors, which are now deprecated, also Vulcanize is explained in the tutorial, and is mentioned in the Polymer 1 documentation, but not Polymer 2

Google Web Fundamentals

Strictly speaking, this is not Polymer specific documentation, this is Google's way of explaining all the new fundamental concepts that Polymer is based upon

This might be too much to read in one go, bet is to keep it as a reference that you can refer to every time you come across a new concept

Polymer Official Docs

Official Polymer Docs, obviously a must read, however bear in mind that it is not really well documented, so this is not a one stop shop for all your needs, unfortunately.

You should also know that Polymer changed a lot between Polymer 1 and Polymer 2, so if you need to work on Polymer 1 then you should read both versions documentation
